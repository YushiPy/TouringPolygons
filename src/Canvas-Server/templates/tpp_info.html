The Touring Polygons Problem Visualizer

This is an interactive visualizer for the Touring Polygons Problem (TPP), a geometric optimization problem. Drag points and polygons around the canvas and watch the shortest path update in real time. This page explains both the underlying theory and how to use the visualizer.

Problem definition

The Touring Polygons Problem (TPP) is a special case of the Traveling Salesman Problem with Neighborhoods (TSPN), which is itself a generalization of the famous Traveling Salesman Problem (TSP). In the TSP, one seeks the shortest route visiting a set of discrete points; the TSPN relaxes this by replacing points with regions, requiring only that the path touch each region rather than a specific location within it. The visit order in the TSPN is also typically left free. The TPP adds two further constraints: the regions must be convex polygons, and the visit order is fixed. These restrictions are precisely what make the problem tractable — without them, Dror et al. proved that the problem is NP-hard.

This problem has applications in robotics navigation, where a robot must pass through a sequence of regions in order, and in logistics, where a route must visit a series of areas in a fixed sequence.

Algorithms and complexity

The TPP was first introduced and studied by Dror et al., who gave the first polynomial-time algorithm for convex polygons:

Dror et al. — Touring a sequence of polygons
Time complexity: O(kn log(n/k)) when polygons are non-intersecting, O(k²n log(n/k)) when they may intersect, where k is the number of polygons and n is the total number of vertices.

A subsequent paper by Tan and Jiang improved upon this result by removing the logarithmic factor:

Tan & Jiang — Efficient Algorithms for Touring a Sequence of Convex Polygons and Related Problems
Presents an O(kn) algorithm for the non-intersecting case, eliminating the log factor from Dror et al.'s result.

This visualizer uses the algorithm from Dror et al., as it is more straightforward to implement. We plan to adopt Tan and Jiang's more efficient algorithm in the future.

The intersecting polygon case has not yet been implemented. The solution displayed may be incorrect if any two polygons overlap. Support for this case is planned.

Non-convex polygons

The TPP for non-convex polygons is NP-hard, as shown by Dror et al., and there has been relatively little work on this case in the literature. We plan to approach it by first computing a convex partition or convex cover of each non-convex polygon, then applying a branch-and-bound search over the resulting convex pieces. This should make it possible to find exact or near-optimal solutions in reasonable time for small inputs.

For now, if any polygon is non-convex, the canvas displays a NOT CONVEX label at its centroid and the solution is suppressed until all polygons are made convex.

Using the visualizer

The canvas is interactive in real time. As you move points or polygons, the shortest path is recomputed and redrawn immediately.

Panning

Click and drag any empty area of the canvas (not on a point or polygon) to pan the view.

Zooming

Scroll the mouse wheel to zoom in or out. Zoom is anchored to the cursor position, so the point under the cursor stays fixed.

Grid

The grid adapts to the zoom level. The main grid spacing is 1, 2, 5, or 10 units depending on zoom. Each main cell is subdivided into 4 or 5 subgrid lines, which are also the snap targets when snapping is enabled.

Saving and loading

Creating an account lets you save and revisit any configuration. Once logged in, the folder icon in the top-left of the header opens the drawings panel, where all your saved drawings are listed and can be searched by name.

Saving a drawing

To save the current configuration, give it a name first — unnamed drawings cannot be saved. Saving an already-named drawing updates it in place rather than creating a new one.

Managing drawings

After selecting a drawing from the panel you can rename it or duplicate it. Duplicating creates a new independent copy, which is useful for exploring variations without overwriting the original.

Points and polygons

Dragging points

Click and drag s, t, or any polygon vertex to move it.

Dragging polygons

Click and drag the interior of a polygon (not on a vertex) to translate the entire polygon at once.

Adding vertices

Click anywhere on the canvas to add a new vertex to the currently active polygon. Clicks are only registered if the mouse has not moved much and the button was released within 300 ms — slow or dragging clicks are ignored.

The vertex is inserted between two consecutive vertices of the active polygon, not appended at the end. Enable the vertex line (button 3 or 3) to preview exactly where it will land before clicking.

Active polygon

Clicks and vertex insertions always target the active polygon. Press ↑ / ↓ to cycle through polygons. The active polygon is visually distinguished from the others.

Adding polygons

Press button 2 or 2 to insert a new equilateral triangle centered on the canvas. Reposition it and add vertices to it as needed. This is the only way to create a new polygon.

Selection

You can select multiple points across any combination of polygons — including s and t — and drag them all together by the same offset. Selection is additive: make multiple passes to accumulate points from different regions.

Creating a selection

Hold Shift and move the mouse to draw a selection rectangle. Release Shift to commit. Repeat to add more points to the selection.

Selected point indicator

Selected points are marked with a rotating dashed ring that animates continuously, so you can always tell what is in your selection even after releasing Shift.

Dragging the selection

Drag any selected point to move the entire selection by the same amount.

Clearing the selection

Double-tap Shift quickly without moving the mouse to deselect all points.

Deleting selected points

Press Backspace, Delete, or x to delete selected points:

  • If all vertices of a polygon are selected, the entire polygon is removed.
  • If only some vertices are selected, deletion stops when 3 remain — a polygon must have at least 3 vertices.
  • s and t are never deleted even if selected.

Toolbar

Snapping 1

Toggles snapping on or off. When enabled, all moved points snap to the nearest subgrid intersection. The button is highlighted when active.

New triangle 2

Inserts a new equilateral triangle centered on the canvas. This is how new polygons are added.

Vertex line 3

Toggles the vertex preview lines. When enabled, two dashed lines are drawn from the cursor to the two vertices that bracket the insertion point, showing exactly where a new vertex would be placed on click.

Resources

The source code for this project, a collection of algorithm implementations, and a detailed technical report (roughly 20 pages) explaining how the algorithms work are all available on GitHub:

YushiPy/TouringPolygons

The report covers the full geometric and algorithmic details behind both the Dror et al. and Tan & Jiang approaches, including proofs and worked examples, and is a good starting point if you want to understand the theory in depth.
